# Правила работы над модулем Tube Power-Amp

Обязательный протокол для ВСЕЙ работы в `src/poweramp/`.
Действует, пока модуль не вынесен в `felitronics-core`.

## 1. Консилиум ПЕРЕД каждым блоком
Перед началом любого блока (см. `PLAN.md`, задачи v1 №1–9) — совещание крю:
- **deepseek** = `deepseek-v4-pro`, `reasoning_effort: max`, **большие лимиты токенов — не жалеть**.
- **codex** = `gpt-5.2-codex`, максимальный effort.

Оба получают одинаковый бриф (контекст блока + что уже сделано + ограничения).
Их ответы я **кросс-чекаю между собой**, и только потом пишу код. Любые расхождения —
выношу тебе на решение, не выбираю молча.

## 2. Консилиум ПОСЛЕ любой разработки
После каждого блока (и любого значимого изменения) — **Claude code review** (`/code-review`)
по дифу. Фиксы из ревью возвращаю в код; при необходимости — повторный прогон.

## 3. Исследования — через ChatGPT
Если нужен ресёрч (модели / библиотеки / лицензии / DSP-теория) — я создаю **`.md`-файл
с промтом** для веб-поиска ChatGPT (он по этой теме ищет лучше codex и меня). Ты запускаешь,
приносишь результат-файл, я свожу его с консилиумом. Ledger лежит здесь же: `oss-research-ledger.md`.

## 4. Язык
Всё, что делаю, **объясняю по-русски**.

## 5. Где живут планы
Все планы / спеки / пересмотры — **только в `src/poweramp/`**. Изоляция: ничего не размазываем
по ветке, пока нет рабочей версии. Минимальный диф в чужих файлах (CMake + один шов в `CabEngine`
за флагом + один параметр + latency/meter).

## Claude (я) в консилиуме
Claude входит в консилиум **не как дубль codex/deepseek** (та же модель → коррелирует), а в отдельных ролях:
- **(всегда) grounding + verify** — отдельный Claude-агент с доступом к репо проверяет, что дизайн
  консилиума бьётся с реальным кодом (точные швы / сигнатуры / тест-харнес / CMake) **до** написания
  кода, и adversarial-ревью дифа **после** блока (`/code-review`).
- **(только тяжёлые блоки) blind independent pass** — отдельный Claude-агент видит **только бриф**
  (без ответов codex/deepseek) → независимое мнение без anchoring; оркестратор сводит вслепую.

Тяжёлый блок помечается явно (например: sag, ООС-петля, виртуальная нагрузка — и любой, что я/ты так
назвали). Скаффолд / механические блоки — обычно только grounding+verify + after-review.
